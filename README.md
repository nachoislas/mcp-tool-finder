# 🤖 Agentic MCP Orchestrator (OpenCode Edition)

> An Autonomous Agent that builds its own tool ecosystem. It doesn't use pre-configured tools — it discovers, installs, and orchestrates them on demand.

---

## The Problem

Current LLM agents depend on pre-configured tools. If the tool doesn't exist, the agent fails. This project changes the paradigm:

```
User: "I need a REST API that connects to my PostgreSQL"
         │
         ▼
   Autonomous Agent
   ┌──────────────────────────────────────┐
   │ 1. Reason: I need DB + API            │
   │ 2. Discover: @modelcontextprotocol/  │
   │              postgres-server          │
   │ 3. Provision: npm install ...       │
   │ 4. Configure: DATABASE_URL=...      │
   │ 5. Execute: create the API          │
   │ 6. Reflect: did it work?           │
   │ 7. Persist: "Prompt → Tools → OK"    │
   └──────────────────────────────────────┘
```

## Vision

An agent that:
- **Reasons** about what tools it needs for each task
- **Discovers** tools in the MCP ecosystem
- **Provisions** (installs/configures) dynamically
- **Executes** complex pipelines
- **Learns** from every success/failure

## Components

### 1. mcp_registry
Catalog of available MCP servers with metadata:

| Field | Description |
|-------|-------------|
| `name` | Package name |
| `capabilities` | What it can do (db, api, filesystem...) |
| `install_cmd` | Installation command |
| `config_schema` | Required variables |
| `rank` | Score based on usage/success |

### 2. agent_memory
Execution history:

| Field | Description |
|-------|-------------|
| `prompt` | Original input |
| `tools_used` | MCPs installed |
| `result` | success/error |
| `duration` | Execution time |
| `tokens_spent` | Token cost |

### 3. skill_store
Code generated by the agent that becomes permanent:

| Field | Description |
|-------|-------------|
| `name` | Skill name |
| `code` | Code (Python/JS) |
| `trigger` | When to run |
| `success_rate` | % of times it worked |

## Agent Flow (Core Loop)

```python
async def agent_loop(user_prompt: str):
    # 1. Ingest
    goal = parse(user_prompt)

    # 2. Planning
    subtasks = decompose(goal)

    # 3. Discovery
    for task in subtasks:
        tools = search_mcp_registry(task, vector_db)
        best_tool = rank(tools).first

    # 4. Provisioning
    if not installed(best_tool):
        await provision(best_tool.install_cmd)
        await configure(best_tool.config)

    # 5. Execution
    result = await execute(best_tool, task)

    # 6. Reflection
    if result.success:
        await index_memory(goal, tools, result)
    else:
        await retry_with_different_tool(task)
```
